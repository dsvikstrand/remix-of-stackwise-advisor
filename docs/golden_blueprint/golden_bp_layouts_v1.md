# Golden BP Layouts v1 (Structure-Only Templates)

## How To Use This File

- Use these as layout scaffolds for generation, not as final prose.
- Keep placeholders during drafting, then fill from transcript evidence.
- Do not copy section text between videos; only reuse the structure.

## Layout A — Fast Value Brief

### Best for
- Videos where readers mainly want quick payoff and key takeaways.

### Target read profile
- Skim: 30-60s
- Full read: 2.5-3.5 min

### Structure

1. `[Title: Outcome-oriented]`
2. `[Summary (primary body): 4-8 short paragraphs or compact bullets]`
<!-- Put key insights, implications, and caveats inside Summary instead of splitting into many sections. -->
3. `[Optional: One Action / Caveat block]`

### Section guardrails
- No section without at least one concrete anchor (number, mechanism, threshold, example).
- No “overview-only” filler language.
- Avoid subtitle-heavy nesting; keep section hierarchy shallow.

---

## Layout B — Deep Dive Operator

### Best for
- Complex topics where depth + credibility matter (research, technical, systems).

### Target read profile
- Skim: 60-90s
- Full read: 3.5-4.5 min

### Structure

1. `[Title: Problem + Shift]`
2. `[Summary (flowing narrative, 220-350 words)]`
<!-- Merge context + thesis into one cohesive block. -->
3. `[Mechanism Deep Dive]`
4. `[Tradeoffs]`
5. `[Decision Rules]`
6. `[Open Questions]`
7. `[Bottom Line]`

### Section guardrails
- Prefer evidence chains over generic explanation.
- Keep jargon only when needed; define briefly when introduced.
- Keep Summary flowing; use later sections for structure.

---

## Layout C — Action Playbook

### Best for
- How-to, workflow, habit, recipe, implementation-heavy videos.

### Target read profile
- Skim: 45-75s
- Full read: 3-4 min

### Structure

1. `[Title: Desired outcome + audience]`
2. `[Summary (flowing practical narrative)]`
<!-- Keep this cohesive and readable; no fragmented mini-headings. -->
3. `[Playbook Steps]`
4. `[Fast Fallbacks]`
5. `[Red Flags / Failure Modes]`
6. `[Bottom Line]`

### Section guardrails
- Every step must include a success signal.
- Avoid abstract motivational language unless tied to a concrete action.
- Keep formatting simple; avoid subtitle clutter.

## Quick Comparison Matrix

| Layout | Best Use Case | Strength | Main Risk | Read Style |
|---|---|---|---|---|
| A — Fast Value Brief | Broad audience, mixed topics | Quick payoff + strong skim value | Can become shallow if evidence is weak | Snappy, high-velocity |
| B — Deep Dive Operator | Technical/research-heavy videos | Depth + trust + mechanism clarity | Can feel dense if not tightly edited | Analytical, structured |
| C — Action Playbook | How-to and execution content | Highest practical transfer | Can feel rigid if over-templated | Practical, task-oriented |

## Selection Hint

- Pick **A** when speed and retention are priority.
- Pick **B** when credibility and nuance are priority.
- Pick **C** when behavior change/action is the priority.
