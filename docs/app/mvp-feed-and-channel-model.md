# MVP Feed And Channel Model

Status: `canonical`

## Purpose
- This document defines the MVP feed model for `For You`, `Joined`, and `All`.
- It is the canonical product contract for how sources, Bleu channels, locked items, and published blueprints should appear across feed surfaces.
- If implementation or older docs conflict with this document, this document wins.

## Core Distinction
- `Sources` are external creators, currently YouTube channels.
- Users `subscribe` to sources.
- `Channels` are Bleu topic lanes such as `b/ai-tools-automation` or `b/cooking-home-kitchen`.
- Users `join` channels.
- Source subscriptions and channel joins are separate systems and must not be treated as the same user action.

## Feed Contracts
### `For You`
- `For You` is the personal source-driven feed.
- It is auth-only.
- It contains:
  - new source items from sources the user is subscribed to
  - blueprint cards created from those subscribed-source items once generation succeeds
  - blueprint cards the user personally unlocked/generated, even when the source is not subscribed
- It is the only feed lane that may show locked items.
- It is latest-first.
- If a subscribed-source item is still locked, it shows as locked.
- If that same item becomes generated by any user, the locked row should resolve to a blueprint card.
- If a user manually unlocks a source they do not subscribe to, that resulting blueprint should appear in that user’s `For You`.
- Future videos from that non-subscribed source do not automatically enter `For You` unless the user later subscribes to the source.

### `Joined`
- `Joined` is the personalized channel-discovery feed.
- It is auth-only.
- It contains only generated and published blueprint cards.
- It is a filtered view of `All`, limited to Bleu channels the user has joined.
- It never shows locked items.
- It is channel-based, not source-based.
- A blueprint appears in `Joined` if:
  - it has been generated
  - it has been published to a Bleu channel
  - the user has joined that Bleu channel

### `All`
- `All` is the global published-blueprint feed.
- It is the aggregation of all generated and published blueprints across Bleu channels.
- It never shows locked items.
- It is the discovery lane for browsing all currently available blueprint content regardless of source subscription state.
- Channel pages such as `b/cooking-home-kitchen` are scoped views inside this same published-blueprint universe.

## Publication Rules
- A blueprint enters `All` or `Joined` only after:
  - generation succeeds
  - Bleu channel classification resolves
  - the blueprint is published to a Bleu channel
- Users do not choose the Bleu channel for MVP.
- Channel assignment is performed by the classifier pipeline.
- If classification is ambiguous, runtime should still resolve to a valid fallback channel such as `general` instead of leaving the blueprint outside the published feed system.

## Locked Item Rules
- Locked items belong only in `For You`.
- `Joined` never contains locked items.
- `All` never contains locked items.
- A locked source item should convert into a blueprint card once generation succeeds.

## Channel Governance
- Bleu channels are admin-curated in MVP.
- Normal users cannot create channels or control classifier routing.
- Users can join channels to personalize `Joined`, but they do not control publication destination.
- This is intentionally different from open community systems like Reddit; the MVP uses a small fixed channel set.

## Community Model
- The app has two parallel discovery systems:
  - source-following through `For You`
  - topic-following through `Joined`
- `All` is the global discovery surface over every published blueprint.
- The long-term community value comes from:
  - likes
  - comments
  - channel/topic browsing
  - finding quality blueprints from sources a user may not already know

## Examples
- If a user subscribes to `Thomas DeLauer` with auto on:
  - new videos enter that user’s `For You`
  - if auto generation succeeds, they appear as blueprint cards
  - if auto generation cannot run yet, they may appear locked until generated
- If a user unlocks a `Doctorly` video without subscribing to `Doctorly`:
  - that resulting blueprint appears in that user’s `For You`
  - future `Doctorly` uploads do not appear unless the user subscribes
- If a blueprint is classified into `b/cooking-home-kitchen`:
  - it appears in `All`
  - it appears in `Joined` for users who joined `b/cooking-home-kitchen`
  - it does not appear in `Joined` for users who did not join that channel

## Non-Goals For MVP
- User-created channels
- User-chosen publish destination
- Locked items in public discovery feeds
- Merging source subscriptions and channel joins into one concept
